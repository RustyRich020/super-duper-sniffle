01

The Personas

Eight roles, each anchored to a surface — the Landing (front door), the Forms Workspace (the work), or the Planner (the roll-up).

SP

Sponsor / Exec

Lives in · Planner
Charter, Business Case
"Is this on track, on budget, worth it?" — reads & approves.
PM

Project Manager

Lives in · Planner (all 8)
Charter, WBS, Schedule, Risk, Status, Change, RACI
Orchestrates everything; the only role in every planner page.
PD

Product Manager

Lives in · Forms · Requirements
PRD, Vision, Roadmap, One-Pager, Stories
Turns intent into requirements the team can build.
EN

Engineer / Tech Lead

Lives in · Forms · Architecture
ADR, Design Doc, README, API Spec, Runbook
Records how & why; keeps living docs alive.
QA

QA / Test Lead

Lives in · Forms · Quality
Test Strategy, Test Cases, UAT, Bug Report
Owns the "done" bar.
OP

SRE / Ops

Lives in · Forms · Operations
Runbook, Incident, Postmortem, On-Call
Operates it when it matters.
CO

Compliance / Auditor

Lives in · Forms + Doc-Gov
SOP, Audit Trail, DPIA, Compliance Matrix
Evidence, control mapping, signoffs.
ST

Stakeholder / User

Lives in · Landing → one form
Intake, requirements input
Contributes once, then watches the dashboards.

The Two Clocks — lifecycle through iterations

A project runs the PMBOK clock once (outer ring). The product runs an iterative loop many times inside Execution (inner ring) — each turn emitting frozen records and refreshing living documents.

Outer ring — the project clock. Five PMBOK phases, traversed once. Each phase ends on an exit artifact: Charter → Baseline → working software → cadence reports → Lessons & Closure.

Inner ring — the product clock. Vision → PRD → Stories → Build → Ship → Learn, repeated every sprint inside Execution.

Frozen records — ADR, RFC, postmortem, CR. Never edited; superseded.
Living documents — README, runbook, risk register. Staleness is a defect.
The Planner is the diff — CPI/SPI trend, risk burn-down, milestone slip, per iteration.
